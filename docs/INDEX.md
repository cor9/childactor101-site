# Project Philosophy

The Child Actor 101 Platform is guided by a few simple ideas.

- Education before marketing.
- Trust before conversion.
- Parents first.
- Children first.
- Build for the long term.
- Teach something on every page.
- Design with clarity.
- Create reusable systems.
- Grow the ecosystem intentionally.

When in doubt, choose the option that builds more trust.

---

# Documentation Standards

All documentation should:

- Be written in Markdown.
- Remain concise and practical.
- Explain *why* as well as *how*.
- Cross-reference related documents.
- Include ownership, status, version, and last updated date.
- Be updated whenever major changes are introduced.

Documentation should evolve alongside the platform.
