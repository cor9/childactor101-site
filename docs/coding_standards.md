# CODING_STANDARDS.md

These standards help maintain consistency across the Child Actor 101 platform.

---

# General Principles

- Write clean, readable code.
- Favor reusable components over duplication.
- Keep components focused on a single responsibility.
- Prioritize accessibility and performance.

---

# Stack

- Next.js App Router
- TypeScript
- Tailwind CSS
- Vercel

---

# Naming

Components:

PascalCase

Example:

Hero.tsx

LessonCard.tsx

Files:

Use descriptive names.

Avoid abbreviations.

---

# Components

Prefer composition over deeply nested components.

Keep files reasonably small.

Extract reusable UI when patterns emerge.

---

# Accessibility

Use semantic HTML.

Support keyboard navigation.

Include descriptive alt text.

Maintain proper heading hierarchy.

---

# Performance

Optimize images.

Avoid unnecessary client-side rendering.

Lazy load media where appropriate.

---

# SEO

Every page should include:

- Metadata
- Open Graph
- Structured data (when appropriate)
- Descriptive page titles
